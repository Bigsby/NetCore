Making changes and committing

The process of making changes and committing them to a shared Git repository is the following:

  1. Make the change

    In Visual Studio Code (VS Code, from now on), open "Program.vs" file and make a change so it looks different, for instance, like this:

    
using System;
namespace ConsoleApplication
{
    public class Program
    {
        public static void Main(string[] args)
        {
            Console.WriteLine("Hello World, I was created by my father, Bigsby, in an unidentified evironment!");
            Console.WriteLine("Bigsby Gates/Trovalds/Jobs was here!");
        }
    }
}

  2. Commit the changes

    Confirm the changes, in the CLI, run:

    $ git status

    The output should state, amongst other things, that "Programs.cs" is modified:

    On branch master
Your branch is up-to-date with 'origin/master'.
Changes not staged for commit:
  (use "git add ..." to update what will be committed)
  (use "git checkout -- ..." to discard changes in working directory)
        modified:   Program.cs
no changes added to commit (use "git add" and/or "git commit -a")

    Stage the chages by running (maybe, one of these days, I'll do a series about Git, but not today):

    $ git add .

    That should have no output. Confirm staging of changes by re-running:

    $ git status

    That should output:

    On branch master
Your branch is up-to-date with 'origin/master'.
Changes to be committed:
  (use "git reset HEAD ..." to unstage)
        modified:   Program.cs

    Commit the staged changes with a message by running:

    $ git commit -m "Add Bigsby Gates/Trovalds/Jobs was here"

    That should output:

    [master cd6ea4f] Add Bigsby Gates was here
 1 file changed, 1 insertion(+)

    One Git command I find very useful in one that shows the delta of commits between branches. This one:

    $ git log --oneline master...origin/master

    That, in this case, should output the commit we just did:

    cd6ea4f Add Bigsby Gates was here

    Finally, push the commit(s) to the remote repository by running:

    $ git push origin master

    That should have an output like so:

    Username for 'https://github.com': bigsbygates
Password for 'https://bigsbygates@github.com':
Counting objects: 4, done.
Compressing objects: 100% (4/4), done.
Writing objects: 100% (4/4), 442 bytes | 0 bytes/s, done.
Total 4 (delta 2), reused 0 (delta 0)
To https://github.com/Bigsby/NetCore.git
   269fa3f..cd6ea4f  master -> master

    Note that authentication against the repository service needs to be provided.
