Creating first .Net Core application

After installing .Net Core, creating a .Net Core application, from the command line, is exactly the same.

The steps to do so:

  1. Create a foldder

    mkdir FirstNetCore

  2. Move into the folder

    cd FirstNetCore

  3. Create the project

    dotnet new

    This will produce an output like so:

    Created new C# project in ...\FirstNetCore.

    Looking into the folder, one finds two files were created:

    • Programs.cs - the program code that looks like this: using System; namespace ConsoleApplication { public class Program { public static void Main(string[] args) { Console.WriteLine("Hello World!"); } } }
    • project.json - the project definition that looks like this: { "version": "1.0.0-*", "buildOptions": { "emitEntryPoint": true }, "dependencies": { "Microsoft.NETCore.App": { "type": "platform", "version": "1.0.0-rc2-3002702" } }, "frameworks": { "netcoreapp1.0": { "imports": "dnxcore50" } } }

      Notice that that is no reference to Programs.cs file project.json. This is because, in the new era of Microsoft projects, it's assumed that all files in the project file's folder and its sub-folders are included in the project. Files can be excluded, but that's for later on...

  4. Get project dependecies

    The project definition, in project.json> file just generated, references, in this case only, one dependency, that needs to be retrieved. For that: dotnet restore

    This should have an output similar to the one found here.

  5. Build the project

    This step is optional because the next step will build if build wasn't been done before. To build:

    dotnet build

    That should have an output similar to this:

    Project FirstNetCore (.NETCoreApp,Version=v1.0) will be compiled because expected outputs are missing Compiling FirstNetCore for .NETCoreApp,Version=v1.0 Compilation succeeded. 0 Warning(s) 0 Error(s) Time elapsed 00:00:01.7172506

  6. Run the application

    Still in the command line

    dotnet run

    That should have an output similar to this if the previous step (build) was performed:

    Project FisrtNetCore (.NETCoreApp,Version=v1.0) was previously compiled. Skipping compilation. Hello World!

    And an output similar to this if previous step (build) was skipped:

    Project FirstNetCore (.NETCoreApp,Version=v1.0) will be compiled because expected outputs are missing Compiling FirstNetCore for .NETCoreApp,Version=v1.0 Compilation succeeded. 0 Warning(s) 0 Error(s) Time elapsed 00:00:01.7172506 Hello World!
