This series tells the story of implementating a web applicatin in .Net Core in 3 (actually, 2.5) Operating Systems:

  • Windows (10);
  • Linux (Debian);
  • Mac OS X (10.11 El Capitan).

For that purpose, the process, from clean Operating System install to full production, will be described. What you see here was added while following the described steps.

The shared techologies and software to be used are:

  • .Net Core (the framework);
  • Visual Studio Code (the IDE);
  • C# programming language (the server side code language);
  • Git (the versioning system) & GitHub (the repository provider);
  • NodeJS (the client side compiling framework) & npm (the client side package manager);
  • Asp.Net Core MVC (the server side framework);
  • TypeScript (the client side code language);
  • Angular 2 (the client side framework);
  • {less} (the client side theming language);
  • Gulp (the client side packaging tool);
  • Windows Azure.

(...these are the ones, will be used, for sure. This list might grow upon usage)

  6. Installing NodeJS (npm)

    Like Git, NodeJS is not needed at this moment but it's, actually, the last install that is not cross-platform so, we'll get the platform specific shenanigans out of the way but installing NodeJS.

    We won't be using NodeJS as a server (although we could, but it defeats the purpose of these series), but will use a component installed with NodeJS called npm that is a package manager, initially intended only for Javascript packages to run in NodeJS but that caters for a lot more, for instance, TypeScript components and references. NodeJS will be used as runtime for running build tasks, mostly, through Gulp...but we'll get there, in time.

    Being NodeJS a software (the last, I promise), installation is different on each platform and, as such, is described in seperate:
